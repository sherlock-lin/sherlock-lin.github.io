文章开始前，想先分享一下学习源码的心得

第一，一定要先用起来，要知道这个组件有什么优缺点，有什么功能特性，这样你就能带着目的去看源码

第二，根据使用该组件的简单案例或者Test进行入口进行调试，切忌直接下载看组件源码

第三，调试的过程中前几遍不要死扣细节，大概清楚流程脉络并绘制流程图，等熟悉后再去攻克细的功能

第四，重要的功能/流程务必要输出文章，这个方式可以强迫你深入思考以及总结归纳，能大大加深理解以及记忆

服务端启动全过程

从以上代码可以看到doBind 才是服务启动的重点方法，咱们在这里看下doBind都具体做了哪些事情：

  1. 调用initAndRegister()初始化并注册Channel，这个方法比较重要

  2. 接下来的regFuture.cause方法判断initAndRegister方法是否发生异常，发生异常直接返回

  3. regFuture.isDone()方法判断initAndRegister是否结束，结束的话调用 doBind0

    如果还没结束的话回注册一个监听器ChannelFutureListener 等初始化结束再调用doBind0

doBind()的实现逻辑很清晰，initAndRegister负责初始化， doBind0 负责监听端口。这两个过程是服务启动的核心

服务端Channel初始化及注册

initAndRegister方法顾名思义就是初始化和注册相关工作，咱们具体看下他的源码实现

initAndRegister具体分为三步：创建Channel、初始化Channel和注册Channel，接下来一步步拆解分析

创建服务端Channel

首先看下创建Channel的过程

通过以上代码可以看到是通过工程类ReflectiveChannelFactory反射创建Channel对象，创建的类是引导器指定的NioServerSocketChannel.class

因此咱们重点关注NioServerSocketChannel构造函数

SelectorProvider 是JDK NIO中的抽象实现，通过 openServerSocketChannel 方法可以创建ServerSocketChannel

SelectorProvider 会根据操作系统的不同返回对应的实现，主流Linux默认返回 EPollSelectorProvider

创建完ServerSocketChannel， 咱们回到NioServerSocketChannel 的构造函数，接着它会通过super依次调用父类的构造方法进行初始化，最终咱们可以定位到AbstractNioChannel 和AbstractChannel 的构造函数

小结
  1. ReflectiveChannelFactory 通过反射创建 NioServerSocketChannel实例
  2. 创建JDK底层的 ServerSocketChannel
  3. 为Channel 创建id、unsafe、pipeline三个重要的成员变量
  4. 设置Channel为非阻塞模式

初始化服务端 Channel

回到initAndRegister方法，继续跟进初始化的init方法

init方法的源码比较长，咱们拆成两个部分看

第一步，设置Socket参数以及用户自定义属性。在创建Channel时 配置参数会保存在NioServerSocketChannelConfig 中，在初始化Channel的过程中，Netty会将这些参数设置到JDK底层的Socket上，并将自定义属性绑定到Channel上

第二步，添加ChannelInitializer处理器，ChannelInitializer是ChannelHandler 的实现类，ChannelInitializer的initChannel方法用于添加其他的Handler。

然后通过异步任务方式向pipeline中创建一个处理器ServerBootstrapAcceptor，专门用于接收新的连接，然后把事件分发给EventLoop 执行

小结

  1. 设置Socket参数以及用户自定义属性

  2. pipeline中添加两个特殊的处理器

注册服务端Channel

Netty会在线程池EventLoopGroup中选择一个EventLoop与当前Channel进行绑定，之后Channel生命周期内的所有IO都由这个EventLoop负责

无论是走哪个逻辑，最终都会调用register0 进行注册

register0() 主要做了四件事：调用JDK底层进行Channel注册、触发handlerAdded 事件、触发channelRegistered 事件、触发channelActive 事件

javaChannel().register() 负责调用JDK底层， 将Channel注册到Selector 上， register() 的第三个参数传入的是Netty自己实现的Channel对象， 调用register方法会将它绑定在JDK底层Channel的attachment上，这样每次selector对象进行事件循环时，Netty都可以从返回的Channel中获得自己的Channel对象

完成注册之后，接下来就是触发Pipeline一系列事件传播。在事件传播之前，用户自定义的业务处理器是怎么添加到Pipeline中的呢？答案就在pipeline.invokeHandlerAddedIfNeeded()当中

可以看出 ChannelInitializer 首先会调用 initChannel()抽象方法，处理完之后会将ChannelInitializer 从Pipeline移出

总结

服务端的启动流程基本为以下四步

  1. 创建服务端Channel

    本质是创建JDK底层原生的Channel，并初始化几个重要的属性，包括id、unsafe、pipeline等

  2. 初始化服务端Channel

    设置参数以及添加两个特殊的处理器 ChannelInitializer 和 ServerBootstrapAcceptor

  3. 注册服务端Channel

    调用JDK底层将 Channel注册到 Selector上

  4. 绑定端口

    调用JDK底层进行端口绑定，并触发channelActive事件，把OP_ACCEPT 事件注册Channel的事件集合中
